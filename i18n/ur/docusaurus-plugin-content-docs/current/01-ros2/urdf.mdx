# Robot Anatomy with URDF

**URDF (Unified Robot Description Format)** ایک XML فائل فارمیٹ ہے جو ROS میں روبوٹ کے تمام عناصر کو بیان کرنے کے لیے استعمال ہوتا ہے۔ یہ ایک بلیو پرنٹ کی طرح ہے جو روبوٹ کی جسمانی ساخت کی وضاحت کرتا ہے۔

## What's in a URDF?

ایک URDF فائل روبوٹ کو اس کے **links** اور **joints** کے لحاظ سے بیان کرتی ہے۔

-   **Links:** یہ روبوٹ کے سخت حصے ہیں (مثال کے طور پر، اوپری بازو، بازو کا اگلا حصہ، ایک پہیہ، چیسس)۔ ہر لنک میں اس کی کمیت، جڑت، اور بصری نمائندگی (یہ کیسا دکھتا ہے) اور تصادم ماڈل (فزکس سمولیشن کے لیے اس کی جسمانی شکل) جیسی خصوصیات ہوتی ہیں۔

-   **Joints:** یہ لنکس کو آپس میں جوڑتے ہیں۔ **Joints** دو لنکس کے درمیان تعلق کی وضاحت کرتے ہیں، بشمول:
    -   **Type:** کیا یہ ایک `revolute` (گھومنے والا) جوائنٹ ہے جیسے کہ کہنی، ایک `prismatic` (سلائیڈنگ) جوائنٹ، یا ایک `fixed` جوائنٹ؟
    -   **Axis:** وہ محور جس کے گرد جوائنٹ حرکت کرتا ہے۔
    -   **Limits:** کم سے کم اور زیادہ سے زیادہ زاویہ یا پوزیشن جو جوائنٹ حاصل کر سکتا ہے۔

### Example URDF Snippet:

```
```xml
<robot name="simple_arm">
  <!-- Base Link -->
  <link name="base_link">
    <visual>
      <geometry>
        <cylinder length="0.6" radius="0.2"/>
      </geometry>
    </visual>
  </link>

  <!-- Arm Link -->
  <link name="arm_link">
    <visual>
      <geometry>
        <box size="0.6 0.1 0.1"/>
      </geometry>
    </visual>
  </link>

  <!-- Joint connecting Base to Arm -->
  <joint name="base_to_arm" type="revolute">
    <parent link="base_link"/>
    <child link="arm_link"/>
    <axis xyz="0 0 1"/>
    <limit effort="300" velocity="1.0" lower="-3.14" upper="3.14"/>
  </joint>
</robot>
```
```
یہ سادہ مثال دو لنکس (`base_link` اور `arm_link`) اور ایک واحد گھومنے والے (`revolute`) جوائنٹ کی وضاحت کرتی ہے جو ان کو جوڑتا ہے۔

## Visualizing with RViz

**RViz (ROS Visualization tool)** ایک طاقتور 3D ویژولائزر ہے جو ایک URDF فائل کو پڑھ سکتا ہے اور آپ کے روبوٹ ماڈل کو ڈسپلے کر سکتا ہے۔ یہ آپ کے روبوٹ کی ساخت کو ڈیبگ کرنے کے لیے ایک ضروری قدم ہے۔

RViz کے ساتھ، آپ یہ کر سکتے ہیں:
-   اپنے روبوٹ کا 3D ماڈل دیکھیں۔
-   تصدیق کریں کہ تمام لنکس درست پوزیشن اور سمت میں ہیں۔
-   روبوٹ کے جوڑوں کو حرکت دینے کے لیے دستی طور پر جوائنٹ سلائیڈرز کو حرکت دیں۔
-   اپنے روبوٹ ماڈل پر دیگر سینسر ڈیٹا (جیسے LIDAR سکین یا کیمرہ فیڈ) کو اوورلے کریں تاکہ کوآرڈینیٹ فریمز کی تصدیق کی جا سکے۔

اس سے پہلے کہ آپ کسی حقیقی روبوٹ پر یا کسی پیچیدہ سمولیشن میں کوڈ چلائیں، آپ کو ہمیشہ RViz میں اپنے URDF کو ویژولائز کرنا چاہیے تاکہ یہ یقینی بنایا جا سکے کہ بنیادی ساخت درست ہے۔
