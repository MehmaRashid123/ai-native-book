# Gazebo میں روبوٹس کی نقالی کرنا

**Gazebo** ایک طاقتور 3D روبوٹکس سمیولیٹر ہے۔ اگرچہ RViz کسی روبوٹ کی حالت کو دیکھنے کے لیے بہترین ہے، لیکن Gazebo وہ جگہ ہے جہاں ہم روبوٹ کے ایک طبعی دنیا کے ساتھ تعامل کی نقالی کر سکتے ہیں۔ اس میں ایک مضبوط فزکس انجن شامل ہے جو کشش ثقل، رگڑ اور تصادم کی نقالی کر سکتا ہے۔

## روبوٹ کو اسپون (Spawn) کرنا

اپنے روبوٹ کو Gazebo میں لانے کے لیے، آپ کو سب سے پہلے ایک URDF فائل کی ضرورت ہے جو اس کی ساخت کو بیان کرتی ہے۔ اس کے بعد آپ اس ماڈل کو Gazebo دنیا میں "اسپون" کرنے کے لیے ایک ROS 2 لانچ فائل استعمال کر سکتے ہیں۔

ایک عام ورک فلو:
1.  **دنیا بنائیں:** ایک `.world` فائل کی وضاحت کریں جو ماحول کو بیان کرتی ہے (مثال کے طور پر، زمینی طیارہ، دیواریں، روشنی)۔
2.  **Gazebo لانچ کریں:** اپنی ورلڈ فائل کے ساتھ Gazebo سرور شروع کریں۔
3.  **اینٹیٹی اسپون کریں:** اپنی URDF فائل کو پڑھنے اور اسے `/robot_description` ٹاپک پر شائع کرنے کے لیے ایک ROS 2 نوڈ (جیسے `spawn_entity.py`) استعمال کریں۔ Gazebo اس ٹاپک کو سنتا ہے اور روبوٹ کو سمولیشن میں رینڈر کرے گا۔

### لانچ فائل کا ایک نمونہ

```
```python
# In a Python launch file
import os
from ament_index_python.packages import get_package_share_directory
from launch import LaunchDescription
from launch.actions import ExecuteProcess

def generate_launch_description():
    # Path to your URDF file
    urdf_file_path = os.path.join(
        get_package_share_directory('my_robot_description'),
        'urdf',
        'my_robot.urdf'
    )

    return LaunchDescription([
        # Start Gazebo
        ExecuteProcess(
            cmd=['gazebo', '--verbose', '-s', 'libgazebo_ros_init.so'],
            output='screen'
        ),

        # Spawn the robot from the URDF file
        ExecuteProcess(
            cmd=['ros2', 'run', 'gazebo_ros', 'spawn_entity.py',
                 '-entity', 'my_robot',
                 '-file', urdf_file_path],
            output='screen'
        ),
    ])
```

## فزکس سیٹ اپ کرنا

Gazebo کی طاقت اس کے فزکس انجن سے آتی ہے۔ اپنی URDF اور SDF (Simulation Description Format) فائلوں میں، آپ اپنے روبوٹ اور دنیا کے لیے اہم طبعی خصوصیات کی وضاحت کر سکتے ہیں:

-   **ماس اور انرشیا:** ہر `<link>` کے لیے، آپ کو اس کا ماس اور انرشیل میٹرکس متعین کرنا چاہیے۔ یہ فزکس انجن کو بتاتا ہے کہ لنک قوتوں اور ٹارک پر کیسے رد عمل ظاہر کرے گا۔
-   **ٹکرانے والے میشز:** `<collision>` ٹیگ فزکس انجن کے لیے ایک لنک کی طبعی شکل کی وضاحت کرتا ہے۔ یہ ایک سادہ شکل (باکس، سلنڈر، اسپیئر) یا ایک پیچیدہ 3D ماڈل (`.stl` یا `.dae` فائل) ہو سکتا ہے۔ فزکس کے حساب کو تیز کرنے کے لیے یہ اکثر `<visual>` میش سے آسان ہوتا ہے۔
-   **رگڑ اور ڈیمپنگ:** ہر جوائنٹ کے لیے، آپ حقیقی دنیا کے میکانزم کی مزاحمت کی نقالی کے لیے رگڑ اور ڈیمپنگ کوایفیشینٹس کی وضاحت کر سکتے ہیں۔
-   **کشش ثقل:** ورلڈ فائل کشش ثقل ویکٹر کی وضاحت کرتی ہے (عام طور پر Z-axis پر -9.8 m/s² )۔

ان پیرامیٹرز کو ٹیون کر کے، آپ ایک ایسی سمولیشن بنا سکتے ہیں جو حقیقی دنیا کے روبوٹ کی طرح برتاؤ کرتی ہے، جس سے آپ طبعی ہارڈ ویئر پر تعینات کرنے سے پہلے اپنے کنٹرول الگورتھم کو محفوظ طریقے سے جانچ سکتے ہیں۔
