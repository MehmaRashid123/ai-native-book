# Setting up NVIDIA Isaac Sim

**NVIDIA Isaac Sim** is a robotics simulation toolkit built on the NVIDIA Omniverse platform. It provides a photorealistic, physics-accurate, and AI-accelerated environment for developing, testing, and training robots.

Unlike Gazebo, which prioritizes fast physics, Isaac Sim prioritizes realism and integration with NVIDIA's AI stack (like Isaac Gym for reinforcement learning).

## System Requirements

Isaac Sim has demanding hardware requirements. Ensure your workstation meets the minimum specs:
-   **OS:** Ubuntu 22.04
-   **GPU:** NVIDIA RTX 2070 or higher (RTX 4070+ recommended). You must use the official NVIDIA proprietary drivers.
-   **CPU:** Intel Core i7 or AMD Ryzen 7 (6-core minimum).
-   **RAM:** 16GB (32GB+ recommended).

## Installation Steps

1.  **Install NVIDIA Omniverse Launcher:**
    -   Go to the [NVIDIA Omniverse website](https://www.nvidia.com/en-us/omniverse/).
    -   Download and install the Omniverse Launcher. This is an application that manages Omniverse apps, including Isaac Sim.

2.  **Install Isaac Sim from the Launcher:**
    -   Open the Omniverse Launcher.
    -   Navigate to the "Exchange" tab.
    -   Search for "Isaac Sim" and click to install it. This will download the simulator, which can be a very large download (5GB+).

3.  **Install Isaac Sim's Python Environment:**
    -   Isaac Sim comes with its own embedded Python environment. You don't need to create a separate virtual environment.
    -   Once installed, you can run Python scripts using the provided `python.sh` wrapper, which ensures all necessary environment variables and paths are set correctly.

    ```bash
    # Example of running a standalone Isaac Sim python script
    ./python.sh my_robot_script.py
    ```

4.  **Enable the ROS 2 Bridge:**
    -   Isaac Sim has built-in support for ROS 2. This is managed through an extension.
    -   Inside Isaac Sim, go to `Window > Extensions`.
    -   Search for `omni.isaac.ros2_bridge` and enable it.
    -   This will automatically bridge ROS 2 topics, services, and actions into the simulation environment, allowing your external ROS 2 nodes to communicate with the simulated robot.

Once set up, you can begin to import your robot's URDF or USD (Universal Scene Description) file, set up a scene, and connect your AI control agents.
