# Capstone Project: The Autonomous Humanoid

This final project is the culmination of everything you have learned. You will integrate all the modules—from ROS 2 fundamentals to VLA-based planning—to create an autonomous humanoid robot capable of understanding and executing a complex, high-level command.

## The Goal

The primary goal is to make the robot perform a "fetch and place" task based on a natural language voice command.

**Example Command:** "Robot, please find the water bottle in the living room and bring it to the kitchen."

## Core Integration Pipeline

To achieve this, you will build a software pipeline that connects all the concepts from this textbook:

1.  **Voice to Text (Module 4):**
    -   A Python node will use **Whisper** to listen for and transcribe your voice command.
    -   **Output:** The text string: `"please find the water bottle..."`

2.  **Text to Plan (Module 4):**
    -   The transcribed text is sent to an **LLM Planner** (e.g., using the OpenAI Agents SDK).
    -   The LLM will decompose the command into a structured JSON plan.
    -   **Output:** A sequence of actions, like:
        ```json
        [
          {"action": "navigate", "location": "living_room"},
          {"action": "find", "object": "water_bottle"},
          {"action": "pickup", "object_id": "bottle1"},
          {"action": "navigate", "location": "kitchen"},
          {"action": "place"}
        ]
        ```

3.  **Plan to Action (Module 1 & 3):**
    -   Your central "execution" node will parse the JSON plan.
    -   For each step, it will call the appropriate ROS 2 Action Server.
    -   `"navigate"` → Calls the **Nav2** stack to move the robot. The robot will use its **SLAM** map to find its way.
    -   `"find"` → Activates a computer vision node to search for the specified object.
    -   `"pickup"` and `"place"` → Trigger complex manipulation sequences, likely pre-programmed trajectories or learned policies.

4.  **Simulation & Control (Module 2 & 3):**
    -   The entire sequence will be run inside **NVIDIA Isaac Sim**.
    -   The simulator will handle the physics of the robot's movement, its interaction with objects, and provide the sensor data (camera feeds, joint states) that the ROS 2 nodes rely on.

## Success Criteria

Your project will be considered a success if you can demonstrate a reliable, end-to-end execution of the task. You should be able to give a voice command and watch the simulated humanoid robot successfully carry out the multi-step instruction.
