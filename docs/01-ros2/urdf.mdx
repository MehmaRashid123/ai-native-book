# Robot Anatomy with URDF

**URDF (Unified Robot Description Format)** is an XML file format used in ROS to describe all elements of a robot. It's like a blueprint that defines the robot's physical structure.

## What's in a URDF?

A URDF file describes the robot in terms of its **links** and **joints**.

-   **Links:** These are the rigid parts of the robot (e.g., the upper arm, the forearm, a wheel, the chassis). Each link has properties like its mass, inertia, and visual representation (how it looks) and collision model (its physical shape for physics simulation).

-   **Joints:** These connect the links together. Joints define the relationship between two links, including:
    -   **Type:** Is it a `revolute` (rotating) joint like an elbow, a `prismatic` (sliding) joint, or a `fixed` joint?
    -   **Axis:** The axis around which the joint moves.
    -   **Limits:** The minimum and maximum angle or position the joint can reach.

### Example URDF Snippet:

```xml
<robot name="simple_arm">
  <!-- Base Link -->
  <link name="base_link">
    <visual>
      <geometry>
        <cylinder length="0.6" radius="0.2"/>
      </geometry>
    </visual>
  </link>

  <!-- Arm Link -->
  <link name="arm_link">
    <visual>
      <geometry>
        <box size="0.6 0.1 0.1"/>
      </geometry>
    </visual>
  </link>

  <!-- Joint connecting Base to Arm -->
  <joint name="base_to_arm" type="revolute">
    <parent link="base_link"/>
    <child link="arm_link"/>
    <axis xyz="0 0 1"/>
    <limit effort="300" velocity="1.0" lower="-3.14" upper="3.14"/>
  </joint>
</robot>
```
This simple example defines two links (`base_link` and `arm_link`) and a single rotating (`revolute`) joint that connects them.

## Visualizing with RViz

**RViz (ROS Visualization tool)** is a powerful 3D visualizer that can read a URDF file and display your robot model. This is an essential step for debugging your robot's structure.

With RViz, you can:
-   See a 3D model of your robot.
-   Verify that all links are in the correct position and orientation.
-   Manually move joint sliders to see how the robot articulates.
-   Overlay other sensor data (like LIDAR scans or camera feeds) onto your robot model to verify coordinate frames.

Before you ever run code on a real robot or in a complex simulation, you should always visualize your URDF in RViz to ensure the fundamental structure is correct.
