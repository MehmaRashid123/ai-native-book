# Python Agents with `rclpy`

`rclpy` is the official Python client library for ROS 2. It allows you to write ROS 2 nodes, publishers, subscribers, and more, directly in Python. This is the primary way we will interface our AI models with the robot's hardware.

## Writing a Simple Publisher

A publisher node sends messages to a topic. Let's create a node that publishes a "Hello World" message.

### The Code (`simple_publisher.py`)

```python
import rclpy
from rclpy.node import Node
from std_msgs.msg import String

class SimplePublisher(Node):
    def __init__(self):
        super().__init__('simple_publisher')
        self.publisher_ = self.create_publisher(String, 'chatter', 10)
        timer_period = 0.5  # seconds
        self.timer = self.create_timer(timer_period, self.timer_callback)
        self.i = 0

    def timer_callback(self):
        msg = String()
        msg.data = f'Hello World: {self.i}'
        self.publisher_.publish(msg)
        self.get_logger().info(f'Publishing: "{msg.data}"')
        self.i += 1

def main(args=None):
    rclpy.init(args=args)
    simple_publisher = SimplePublisher()
    rclpy.spin(simple_publisher)
    simple_publisher.destroy_node()
    rclpy.shutdown()

if __name__ == '__main__':
    main()
```

### Breakdown:
1.  We import the necessary libraries, including `Node` and the `String` message type.
2.  Our `SimplePublisher` class inherits from `Node`.
3.  In the constructor, we call `create_publisher()` to create a publisher that sends `String` messages on the topic named `chatter`.
4.  We create a timer that calls `timer_callback` every 0.5 seconds.
5.  The `timer_callback` function creates a `String` message, populates it, and publishes it.

## Writing a Simple Subscriber

A subscriber node listens for messages on a topic. Let's create a node that listens to our `chatter` topic.

### The Code (`simple_subscriber.py`)

```python
import rclpy
from rclpy.node import Node
from std_msgs.msg import String

class SimpleSubscriber(Node):
    def __init__(self):
        super().__init__('simple_subscriber')
        self.subscription = self.create_subscription(
            String,
            'chatter',
            self.listener_callback,
            10)
        self.subscription  # prevent unused variable warning

    def listener_callback(self, msg):
        self.get_logger().info(f'I heard: "{msg.data}"')

def main(args=None):
    rclpy.init(args=args)
    simple_subscriber = SimpleSubscriber()
    rclpy.spin(simple_subscriber)
    simple_subscriber.destroy_node()
    rclpy.shutdown()

if __name__ == '__main__':
    main()
```

### Breakdown:
1.  We initialize a node named `simple_subscriber`.
2.  We call `create_subscription()` to subscribe to the `chatter` topic.
3.  We tell ROS 2 to call our `listener_callback` function whenever a new message arrives.
4.  The `listener_callback` function simply logs the data it received to the console.

`rclpy.spin()` is the magic that keeps the node running and processing callbacks until you shut it down (e.g., with Ctrl+C).
